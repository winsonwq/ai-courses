# AI Agent 命名与路径规范

## 路径字符问题与规则更新

根据提交 `bde74a6231ed987a8f5f79e75a9754c9788a22fe` 的修复内容，我们在跨平台开发中发现特殊字符会导致严重的 Git 路径问题（如 `error: invalid path`）。因此制定以下强制规则：

### 核心规则

1. **严禁使用冒号**
   - **禁止**：使用中文冒号 `：` 或英文冒号 `:` 作为文件名或目录名的一部分。
   - **原因**：Windows 系统和某些文件系统不支持冒号，导致 `git clone` 或 `checkout` 失败。
   - **替代方案**：使用连字符 `-` 或空格代替。

2. **字符安全范围**
   - 推荐仅使用：
     - 字母 (A-Z, a-z)
     - 数字 (0-9)
     - 连字符 (`-`)
     - 下划线 (`_`)
     - 必要的空格 (但在纯代码项目中建议避免空格)

### 案例说明

| 错误示例 (禁止) | 正确示例 (推荐) |
| :--- | :--- |
| `L04 AGENTS.md：AI Agent` | `L04 AGENTS.md-AI Agent` |
| `Chapter 1: Intro` | `Chapter 1-Intro` |
| `Skill “Search”` | `Skill-Search` |

> **注意**：所有新建的 Agent 配置文件、技能目录及文档均需严格遵守此规范，以确保项目在 Mac, Linux, Windows 上的兼容性。
